# Реализация автоматического обновления данных

## Что было сделано

### 1. Автообновление в лаунчере (Launcher/src/App.tsx)
- Добавлен интервал автоматического обновления данных пользователя каждые 30 секунд
- При каждом обновлении загружаются актуальные данные с сервера (подписка, статус бана, аватарка)
- Данные автоматически синхронизируются с localStorage
- Обновление происходит незаметно для пользователя без перезагрузки страницы

### 2. Автообновление на сайте (src/pages/DashboardPage.tsx)
- Добавлен интервал автоматического обновления данных пользователя каждые 30 секунд
- Обновляются все данные профиля включая аватарку
- Изменения подписки или бана сразу отображаются в интерфейсе

### 3. Автообновление в админ-панели (src/pages/AdminPage.tsx)
- Добавлен интервал автоматического обновления списка пользователей и новостей каждые 30 секунд
- Администратор видит актуальные данные без необходимости обновлять страницу

### 4. Удаление кнопок управления аватаркой из лаунчера
- Из лаунчера удалены кнопки "Восстановить гугл аватарку" и "Изменить аватарку"
- Пользователи лаунчера могут управлять аватаркой только через веб-версию
- В веб-версии (дашборде) кнопка "Изменить аватар" осталась

## Технические детали

### Интервал обновления
- Частота: каждые 30 секунд (30000 мс)
- Можно изменить, отредактировав значение в `setInterval`

### Что обновляется автоматически
- Подписка пользователя (free/premium/alpha)
- Статус бана (isBanned)
- Аватарка пользователя
- Все остальные данные профиля
- Список пользователей в админ-панели
- Список новостей в админ-панели

### Очистка ресурсов
- При размонтировании компонентов интервалы автоматически очищаются через `clearInterval`
- Это предотвращает утечки памяти

## Преимущества
1. Пользователи сразу видят изменения подписки или бана
2. Не требуется перезагрузка страницы
3. Обновление происходит незаметно в фоновом режиме
4. Администраторы видят актуальные данные в реальном времени
5. Упрощенное управление аватаркой - только через веб-версию

## Примечания
- Автообновление работает только когда пользователь залогинен
- При ошибках сети обновление пропускается, но попытки продолжаются
- Логи обновлений выводятся в консоль для отладки
